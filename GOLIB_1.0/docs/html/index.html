GOLIB provides a collection of C++ classes for global optimization and guaranteed parameter estimation of algebraic and dynamic models.

On distribution, GOLIB should contain four directories: src, include, examples, and docs. The library comes with a Doxygen documentation, which can be accessed by opening the file docs/html/index.html with any html interpreter. Also, a number of simple examples is provided in the examples subdirectory that should help the user understand how to use the various C++ classes. GOLIB consists of header files only (no need to build/link a library), which can be found in the src directory.

Installing GOLIB

cd into the src directory, and type

> make install

to create symbolic links to the header files in the include directory. Note that the code should compile without warning using gcc version 4.6 or higher.

In order to remove all symbolic links created in the include directory, type

> make cleaninstall

from the src directory.

Important Notes: GOLIB relies on a number of third party libraries.

  • MC++ for bounding and relaxation of factorable functions
  • ODEBND for bounding of the solutions of parametric initial value problems in ODEs
  • Solution of polyhedral relaxations of factorable programs relies on either CPLEX or GUROBI.
  • Numerical solution of NLP and IVP relies on the libraries IPOPT and SUNDIALS/CVODES, respectively.

All these libraries must be installed prior to using GOLIB, and the paths to these libraries and header files are to be changed in the first section of the makeoptions.mk file in the src directory.

Running GOLIB

Change into the examples directory, and select one of the subdirectories, e.g.

> cd examples/MC-1D

Type:

> make

at the command line, to create the executable file exm_1D. This executable can then be run by typing:

> ./MC-1D

This executable generate a file containing the relaxations and subgradients. These results can be displayed with the program GNUPLOT by typing:

> gnuplot MC-1D.g
